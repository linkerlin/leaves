# 记忆宪法 (Memory Constitution)

## 四条公理

### 公理一：验证优先 (Action-Verified Only)
只有经过工具执行验证的信息才能写入记忆。
未经验证的 LLM 推测、假设、计划不得写入任何记忆层。

### 公理二：数据神圣 (Sanctity of Verified Data)
已验证数据不可被未验证信息覆盖。
只有执行验证后的新信息才能替换旧信息。

### 公理三：无易变 (No Volatile State)
不存储易变状态：当前时间、临时变量、会话特定计数器。
只存储跨会话持久的知识。

### 公理四：最小指针 (Minimum Sufficient Pointer)
索引层（L1）只存最小指针，不存内容。
内容存储在 L2/L3，通过 L1 索引按需加载。

## 信息分类决策树

1. 信息是否经过工具执行验证？
   - 否 → 不写入，返回
   - 是 → 继续
2. 信息是否跨会话持久有用？
   - 否 → 不写入（易变状态）
   - 是 → 继续
3. 信息属于哪一类？
   - 环境/路径/配置 → L2 事实层
   - 可复用流程/SOP → L3 SOP 层
   - 项目事实/决策 → L2 事实层
   - 偏好/红线/约束 → L2 事实层（标记为 RULE）
4. 是否需要更新 L1 索引？
   - 新增了 L2/L3 条目 → 是，更新 L1 指针
   - 仅更新已有条目内容 → 否

## 各层写入权限

| 层级 | 谁可以写入 | 写入条件 | 格式 |
|------|-----------|---------|------|
| L0 宪法 | 仅人工编辑 | 无 | Markdown |
| L1 索引 | Agent（通过 start_long_term_update） | L2/L3 变更时同步更新 | ≤30 行纯文本 |
| L2 事实 | Agent（通过 start_long_term_update） | 经过工具验证 | Markdown sections |
| L3 SOP | Agent（通过 start_long_term_update） | 执行路径验证成功 | Markdown + 可执行脚本 |
| L4 归档 | 系统（自动） | 会话完成时 | 压缩 JSONL |
